What is REST?

REST

Representational State Transfer

Standardized architectural style for designing networked applications.

It uses HTTP protocols (GET, POST, PUT, DELETE, ...) to interact with resources.

Key Features of REST

  • Stateless
  • Client-Server Architecture
  • Cacheable
  • Uniform Interface
  • Layered System

Request:


              Host: api.example.com

              GET:
                   URI: /users/123
                   URI: /users/123/posts
              
              URL: http:/api.example.com/users/123 
              URL: http:/api.example.com/users/123/posts

Response:


            GET: /user/123
                {
                  "user": {
                    "id": 123,
                    "name": "John Doe",
                    "address: {...}",
                  }
                }

            GET: /user/123/posts
                {
                  "posts": [
                    {
                      "id": ncjd823,
                      "title": "...",
                      "content": "...",
                    }
                  ],
                }

Pros & Cons of REST


  • ✅ Simplicity
  • ✅ Scalability
  • ✅ Caching Supported
  • ❌ Over/Under Fetching
  • ❌ Multiple Round-Trips

What is GraphQL?

GraphQL is an open-source data query language for APIs

It is a server-side runtime for executing the query.

Key Features of GraphQL

  • Flexible Queries
  • Single Endpoint
  • Strongly Typed Schema
  • Mutations
  • Introspection
  • Subscriptions (Real-Time Data)

Request:


              URL: api.example.com/graphql

              HTTP POST:
                  query {
                    user(id: "123") {
                      id
                      name
                      post {
                        id
                        title
                      }
                    }
                  }

Response:


            HTTP POST:
                {
                  "data": {
                    "user": {
                      "id": "123",
                      "name": "John Doe",
                      "posts": [
                        {
                          "id": "ncjd823",
                          "title": "...",
                        }
                      ]
                    }
                  }
                }

Pros & Cons of GraphQL


  • ✅ Flexible Queries
  • ✅ Fewer Requests
  • ✅ Real-Time Capabilities
  • ❌ Complex Caching
  • ❌ Overhead for Small APIs

REST / GraphQL


  • Fixed Endpoints / Flexible Queries
  • Multiple Requests / Single Request
  • Over-fetching / Precise Data

When to choose which one?

REST

  • You need simplicity, caching, or wide compatibility
  • Your data structure is stable and predictable

GraphQL

  • You have complex frontend requirements
  • You want to reduce network requests and avoid over-fetching
  • Real-time updates (subscriptions)
